# AI Agent Development Blueprint: `jref alias`

## Objective
Implement a highly robust, recursive command aliasing engine for the `jref` CLI tool, capable of expanding user-defined shorthand commands into complex sequences of built-in commands and flags. 

## Architectural Context & Constraints
* **Entry Point Interception:** The alias resolution must occur in `src/index.ts` immediately after `parseGlobalOptions` but *before* `registry.get(commandName)` is called.
* **Storage Protocol:** Aliases should be persisted in a `.jref/config.json` file. Support both local (cwd) and global (`~/.jref/config.json` or equivalent) scopes.
* **Recursion & Safety:** Aliases may reference other aliases. You MUST implement a cyclic dependency guard (a stack trace of traversed keys) that throws a fatal error if an infinite loop is detected.

## Implementation Steps
1. **Types & Schema (`src/types/index.ts`):** * Define an `AliasConfig` interface and a strict `zod` schema to validate the alias configuration file structure.
2. **Alias Resolver (`src/utils/alias.ts`):** * Write an `expandAliases(args: string[], config: AliasConfig): string[]` function.
   * If `args[0]` matches a key in the alias map, replace it with the mapped array of strings and recursively process the new array.
   * Ensure any trailing arguments provided by the user are correctly appended to the expanded alias.
3. **Command Implementation (`src/commands/alias.ts`):**
   * Build the CLI command to manage aliases (e.g., `jref alias set <name> <command...>`, `jref alias remove <name>`, `jref alias list`).
   * Add flags for `--global` and `--local` to determine where the alias is saved.
4. **Integration (`src/index.ts`):**
   * Import the alias resolver and modify the `remainingArgs` array before looking up the command in the registry.

## Debugging Protocol
The primary user develops in constrained environments (e.g., Termux on Android, headless Raspberry Pi 4). Console-based debugging and interactive terminal windows are highly inaccessible. 
* **DO NOT** prompt the user to add `console.log` statements for debugging.
* **MUST** implement internal file-based logging (e.g., writing error traces to `.jref/debug.log`) for any parsing failures or circular dependency errors.
