# AI Agent Development Blueprint: `jref git` / `jgit`

## Objective
Integrate a virtualized Git abstraction layer that allows users to traverse, analyze, and mutate Git trees and commit histories directly within the `jref` serialized snapshot paradigm, accompanied by a JSON-capable TUI.

## Architectural Context & Constraints
* **Cross-Architecture Compatibility:** The primary deployment targets include ARM-based devices (Raspberry Pi 4, Termux Android). Native C++ bindings (like `nodegit`/`libgit2`) routinely fail compilation here. You **MUST** strictly utilize pure JavaScript implementations like `isomorphic-git` or use `child_process` wrappers like `simple-git` mapped to the system's local Git binary.
* **TUI Foundation:** Leverage the existing `ink` and `react` dependencies to build the interactive interface. 

## Implementation Steps
1. **Dependency Integration (`package.json`):**
   * Add `isomorphic-git` and `memfs` (or similar virtual filesystem layer) to handle Git operations strictly in-memory against the snapshot's state.
2. **Command Logic (`src/commands/git.ts`):**
   * Implement subcommands for tree traversal, diff generation, and commit history parsing.
   * Ensure every Git operation can output in strict JSON, YAML, TOML, or XML (respecting the `jref` `--json` or `-s` flags) to support piping and automation.
3. **Interactive TUI Component (`src/components/GitUI.tsx`):**
   * Build a modular React/Ink TUI for interactive staging and commit management. 
   * Ensure keybinds map correctly to constrained environments (e.g., avoiding complex multi-key modifiers that fail on mobile terminal keyboards).

## Debugging Protocol
* TUI environments completely swallow standard console errors. All `isomorphic-git` promise rejections and filesystem mapping errors MUST be caught and appended to `.jref/debug.log` with complete stack traces.
