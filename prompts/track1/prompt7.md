# Role and Objective
You are an expert TypeScript/Node.js CLI developer working on `jref`, a tool designed to create and interact with condensed project snapshots. 

Your current task is to drastically expand the capabilities of the `pack` command (`src/commands/pack.ts`) by exposing several advanced features of the underlying `repomix` programmatic API. Currently, the `pack` command hardcodes several optimal features to `false` or restricts them to standard `json` defaults. Your goal is to map new CLI flags to these internal configurations.

# Context: Current Implementation
The `PackCommand` currently handles both local directory packing (via `pack()`) and remote repository packing (via `runRemoteAction()`). 
- It utilizes an internal `config` object for local execution.
- It utilizes a `cliOptions` object for remote execution.
- It currently hardcodes `style: 'json'` and `compress: false`.
- It implements custom chunking using a `--max-size` flag, which wraps the `repomix` output in a custom `ProjectSnapshot` interface.

# Required Feature Additions

Please implement the following updates by modifying the `PackFlags` interface, the `CommandDefinition` options array, the `parseArgs` parser, and the execution logic where `repomix` is invoked.

## 1. Multi-Format Output Support (Markdown & XML)
`repomix` natively supports outputting repositories into formatted Markdown or XML, which is highly optimized for varying LLM context windows.
- **Task:** Introduce an `--output-style <json|markdown|xml|plain>` flag (alias `-s`).
- **Integration:** - Map this to `config.output.style` for local packing and `cliOptions.style` for remote packing.
  - **Crucial Architecture Note:** `jref` natively relies on JSON structure (`createSnapshot` constructs a `ProjectSnapshot` object). If the user selects `--output-style markdown` or `--output-style xml`, bypass the custom `ProjectSnapshot` wrapper and directly output the raw string result generated by `repomix`. Only utilize the `createSnapshot` wrapper and the chunking (`--max-size`) logic if the style is `json` (the default).

## 2. Advanced Remote Packing Configurations
The current remote execution logic (`runRemoteAction`) works but lacks granular control for the user.
- **Task:** Introduce the following flags for remote targeting:
  - `--branch <name>`: Target a specific branch.
  - `--commit <hash>`: Target a specific commit.
- **Integration:** Append these parameters to the target URL or the specific configuration schema required by `runRemoteAction` to ensure precise repository state fetching.

## 3. Token Optimization & Compression Algorithms
To maximize context efficiency for AI agents, we need to expose `repomix`'s native compression and stripping capabilities.
- **Task:** Introduce the following boolean flags and value flags:
  - `--compress`: A boolean flag to enable `repomix`'s source code compression (removing unnecessary whitespace while preserving AST validity). Map to `output.compress`.
  - `--remove-comments`: A boolean flag to strip code comments from the output. Map to `output.removeComments`.
  - `--remove-empty-lines`: A boolean flag to strip blank lines. Map to `output.removeEmptyLines`.
  - `--top-files-length <number>`: A numeric flag to limit the number of top-level files processed. Map to `output.topFilesLength`.
  - `--token-limit <number>`: Map to `config.output.tokenCountTree` or specific token constraints if applicable within the `repomix` API, ensuring we cut off the pack before blowing out context limits.

# Implementation Requirements
1. **Type Safety:** Ensure all new flags are properly typed within the `PackFlags` interface.
2. **Immutability of Existing Features:** Do not break the existing `--max-size` (chunking), `--instruction`, or `--summary` logic when `--output-style` is set to `json`.
3. **Graceful Fallbacks:** If the user specifies an unsupported combination (e.g., `--max-size` chunking alongside `--output-style xml`), emit a clear warning to `stderr` that chunking is only supported for JSON outputs, and proceed without chunking or safely exit.
4. **Verbosity:** Update the `workflows` array in the `CommandDefinition` to include detailed explanations of how the new optimization flags reduce token overhead for LLMs.

Please provide the complete, refactored `src/commands/pack.ts` file, prioritizing clean logic separation between the flag parsing and the `repomix` configuration payload mapping.
