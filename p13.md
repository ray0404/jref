# AI Agent Development Blueprint: `jref tool` / `jtool`

## Objective
Develop a CLI tool orchestrator designed to execute standard POSIX tools, intercept their `stdout`/`stderr` streams, and normalize the output into strictly structured, schema-validated JSON for downstream processing or AI agent ingestion.

## Architectural Context & Constraints
* **Process Streaming:** Utilize Node's `child_process.spawn` to ensure memory-efficient streaming of large tool outputs, rather than buffering entire outputs into memory via `exec`.
* **Interoperability:** The normalized JSON output must be directly pipeable into `jref query` or exposed via the `jref serve` MCP protocol.

## Implementation Steps
1. **Core Engine (`src/utils/tool-runner.ts`):**
   * Implement a spawn wrapper that executes a target command (e.g., `ls`, `ping`, `ifconfig`).
   * Capture the stream buffers and handle process exit codes securely.
2. **Parser Registry (`src/parsers/`):**
   * Establish a plugin-like architecture for text-to-JSON parsers (similar to the Python `jc` utility). 
   * Create an initial set of parsers for standard utilities (e.g., mapping `ls -la` output into an array of objects with `permissions`, `owner`, `size`, `filename` properties).
3. **Command Implementation (`src/commands/tool.ts`):**
   * Parse the user input, route the text stream to the appropriate parser, and output the serialized JSON data structure.

## Debugging Protocol
* When an external tool fails to execute or a parser fails to match the text blob schema, do not crash the pipeline. Output a standardized JSON error object to `stdout` containing the raw output, and silently write the parsing stack trace to `.jref/debug.log`. Do not request debugging output from the user.
