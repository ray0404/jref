# AI Agent Development Blueprint: `jref bin` / `jbin`

## Objective
Develop a command executor that allows users to run scripts embedded directly within serialized `.json` project snapshots without extracting them to the local filesystem, utilizing a dynamic global/local path resolution strategy.

## Architectural Context & Constraints
* **Path Resolution:** Instead of relying on a hardcoded local file path, the command must scan paths defined in a `$JREF_BIN_PATH` environment variable, defaulting to local `.jref/bin/` and global `.jref/bin/`.
* **Termux Environment Specifics:** The target environments include Termux on Android. The standard global binary path in Termux is strictly `/data/data/com.termux/files/usr/bin`.
* **Argument Isolation:** Ensure strict demarcation between `jref` execution flags and the embedded script's arguments. The `--` token must be used to cleanly slice `process.argv`.

## Implementation Steps
1. **Command Implementation (`src/commands/bin.ts`):**
   * Register the `bin` command in the registry.
   * Parse the target `[FILE]` and `[COMMAND/SCRIPT]`.
   * Traverse `$JREF_BIN_PATH` or the default `.jref/bin/` directories to locate the target `.json` snapshot.
2. **Execution Wrapping:**
   * Extract the specific script from the snapshot's virtual filesystem using the existing logic found in `jref run`.
   * Pass the correctly sliced arguments (everything after `--`) into the isolated execution context.
3. **Symlink Generation Utility:**
   * Create an auxiliary setup function that can generate symlinks in `/data/data/com.termux/files/usr/bin` (or standard `/usr/local/bin` for Linux) that point directly to the `jref bin` execution wrapper, allowing the user to call `jbin <command>` globally.

## Debugging Protocol
* Suppress noisy standard output. If the executable snapshot is missing or corrupted, fail silently to `stdout` but write a detailed diagnostic trace to `.jref/debug.log`. Do not ask the user to fetch console outputs.
